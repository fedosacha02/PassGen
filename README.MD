All passwords are organised in folders.
A shared password is one that mates (peers) can view their name and value but cannot change or delete it. 
If a user cancels password sharing to a group the password disapeares from all organisations where they have been shared.
A user can:
1. create a password (up to 10 passwords);
2. rename a password;
3. create a group (up to 3 groups);
4. place a password in different folders (up to 3 folders);
5. change the master password;
6. ask for joining in an existing group.
7. delete the account;

When a user joins another group, they also becomes a group member.
A group member can:
1. request for the sharing their password to the group;
2. unshare an own password from a group;
3. leave the group.


A user may create a group also becoming a group admin.

A group admin can:
1. invite, accept and delete members from the group (form the group) (up to 5 members);
2. accept the request for the sharing a password from a member (up to 3 shared passwords); 
3. make a password unshared;
4. reject the request for the sharing a password from a member (up to 3 shared passwords);
5. rename the group;
6. delete the group;
7. place a shared password in different folders (up to 3 folders).
8. create an organisation (up to 1 organisation).
9. ask for joining the group in an existing organisation;
10. make the group leave an organisation.

When a group joins an organisation they also becomes a group-member of this organisation.
All the members of a group-member can access all passwords shared to their organisation.

A group admin may create an organisation (up to 1 organisation) becoming also a superadministrator. 
Their group members become ordinal organisation administrators. The group can then be called an administrative group.

A superadministrator administrates both the group and the organisation. They can:
1. invite (up to 4 ordinal groups), accept and delete ordinal groups from the organisation (form the organisaition);
2. form the administrative group;
3. place a shared password in different folders (up to 3 folders);
4. accept the request for the sharing a password from a group-member admin (up to 3 x 4 = 12 shared passwords for the all organisation);
5. reject the request for the sharing a password from a group-member admin;
6. make an organisation password unshared;
7. rename the organisation;
8. delete the organisation.

Ordinal organisation administrators of the administrative group can:
1. accept the request for the sharing a password from a group-member admin (up to 3 x 4 = 12 shared passwords for the all organisation);
2. reject the request for the sharing a password from a group-member admin;
3. make an organisation password unshared;
4. place a shared password in different folders (up to 3 folders).
